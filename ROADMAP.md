# Roadmap

Soul's roadmap is ordered by evidence, not feature count.

## Current: 4.0.x — make the trust boundary easy to adopt

- reliable installation and upgrade path;
- visible CI, security reporting and contributor workflow;
- clearer examples for durable runs and declarative skills;
- no changes to the 23-tool surface without a demonstrated user problem.

## Next candidates

- run, receipt, episode and skill metrics in `soul_status`;
- a sectioned PassportEnvelope writer matching the existing fail-closed reader;
- verified starter skills with reproducible evidence, not a large unreviewed
  marketplace;
- measurement of retrieval changes against real, consented query feedback;
- the preregistered model-evaluation wave already described in `eval/`.

These are candidates, not release promises. Each needs its own compatibility,
privacy and causal-evidence gate.

## Explicit non-goals

- a hosted account or mandatory cloud sync;
- silently collecting prompts or telemetry;
- autonomous process execution inside the memory server;
- declaring model capability gains from infrastructure alone;
- maximizing tool count at the cost of a stable contract.
